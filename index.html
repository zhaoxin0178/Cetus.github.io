《JS高级》阅读笔记——第四章

发表于 2019-03-06

基本类型和引用类型的值

基本类型值指简单的数据段，可以操作保存在变量中的实际的值。
引用类型值指那些可能由多个值构成的对象，保存在内存中。Javascript不允许直接访问内存中的位置，因此引用类型值按引用来访问。

动态的属性

定义基本类型值和引用类型值：创建一个变量并为该变量赋值。

  1. 对于引用类型的值，可以添加属性和方法，也可以改变和删除属性和方法。
  2. 基本类型值不能添加属性和方法。

复制变量值

  1. 从一个变量向另一个变量复制基本类型的值时，会在变量对象上创建一个新值，然后把该值复制到为新变量分配的位置上。即复制的只是一个副本，两个变量的值之间不会相互影响。
  2. 当从一个变量向另一个变量复制引用类型的值时，同样会将存储在变量对象中的值复制一份放到为新变量分配的空间中。不过这个值的副本实际上是一个指针，这个指针指向存储在堆中的一个对象。复制后两个变量将引用同一个对象，因此改变其中一个变量，会影响另一个变量。

传递参数

ECMAScript中所有函数的参数都是按值传递的。

  1. 在向参数传递基本类型的值时，被传递的值会被赋值给一个局部变量。
  2. 在向参数传递引用类型的值时，会把这个值在内存中的地址复制给一个局部变量，因此这个局部变量的变化会反映在函数的外部。

检测类型

  1. typeof操作符可以确定一个变量是字符串、数值、布尔值、undefined值还是对象。
  2. instanceof操作符可以确定某个对象的具体类型。

执行环境及作用域

执行环境定义了变量或函数有权访问的其他数据，决定了它们各自的行为。每个执行环境都有一个与之关联的变量对象，环境中定义的所有变量和函数都保存在这个对象中。全局执行环境被认为是window对象，因此所有全局变量和函数都是作为window对象的属性和方法创建的。某个执行环境中的所有代码执行完毕后，该环境被销毁，保存在其中的所有变量和函数定义也会随之销毁。
当代码在一个环境中执行时，会创建变量对象的一个作用域链。作用域链的用途是保证执行环境有权访问的所有变量和函数的有序访问。作用域链的前端是当前执行的代码所在环境的变量对象（如果是函数则将活动对象作为变量对象），下一个变量对象来自外部包含环境，再下一个变量对象来自下一个包含环境，一直延续到全局执行环境。
标识符解析是沿着作用域链一级一级地搜索标识符的过程。搜索过程始终从作用域链的前端开始，然后逐级向后回溯，直到找到标识符为止。

延长作用域链

为了延长作用域链，可以在作用域链的前端临时增加一个变量对象，该变量对象会在代码执行后被移除，包括以下两种语句：

  1. try-catch语句的catch块。
    catch语句会创建一个新的变量对象，其中包含的是被抛出的错误对象的声明。
  2. with语句。
    with语句会将指定的对象添加到作用域链中。

with语句接收了location对象，因此其变量对象中包含了location对象的所有属性和方法。当在with语句中引用href时，实际上引用的是location.href。

没有块级作用域

JavaScript没有块级作用域，即在if、for等语句中创建的变量会存在于其外部的环境中。

  1. 声明变量
    使用var声明的变量会自动被添加到最接近的环境中。
    如果初始化时没有使用var声明，该变量会自动被添加到全局环境。
  2. 查询标识符
    搜索标识符的过程从作用域链的前端开始，向上逐级查询与给定名字匹配的标识符。如果局部环境中找到了该标识符，搜索过程停止，不会再向上查询。如果局部环境中没有找到该变量名，则继续沿作用域链向上搜索，知道全局环境的变量对象。

垃圾收集

JavaScript具有自动垃圾收集机制，即执行环境会负责管理代码执行过程中使用的内存。原理是找出那些不再继续使用的变量，然后释放其占用的内存。

标记清除

当变量进入环境时，就将这个变量标记为“进入环境”。而当变量离开环境时，则将其标记为“离开环境”。垃圾收集器在运行的时候会给存储在内存中的所有变量都加上标记。然后它会去掉环境中的变量以及被环境中的变量引用的变量的标记。在此之后再被加上标记的变量将被视为准备删除的变量，因为环境中的变量以及无法访问到这些变量。最后，垃圾收集器完成内存清除工作，销毁那些带标记的值并回收内存空间。

引用计数

引用计数的含义是跟踪记录每个值被引用的次数。当声明了一个变量并将一个引用类型值赋给该变量时，则这个值的引用次数为1。如果同一个值又被赋给另一个变量，则该值的引用次数加1。相反，如果包含这个值引用的变量取得了另一个值，则原值的引用次数减1。当该值的引用次数变成0时，回收其内存空间。
但引用计数会导致循环引用问题，使一些相互引用的对象永远不能被清除。

性能问题

垃圾收集器是周期运行的，而且如果为变量分配的内存数量很多，那么回收工作量也会很大。因此，确定垃圾收集的时间间隔是一个非常重要的问题。

管理内存

优化内存占用的最佳方式，就是采用解除引用的做法，为执行中的代码只保存必要的数据。一旦数据不再有用，通过将其值设置为null的方式，让该值脱离执行环境，以便垃圾收集器运行时将其回收，释放内存。

对象

对象：无序属性的集合，其属性可以包含基本值、对象或函数。

属性类型

ECMA-262第5版在定义只有内部才用的特性时，描述了属性的各种特征，这些特性在JS中不能直接访问。

  1. 数据属性
    • [[Configurable]]：表示能否通过delete删除属性，能否修改属性的特性，能否把属性修改为访问器属性。
    • [[Enumerable]]：表示能否通过for-in循环返回属性。
    • [[Writable]]：表示能否修改属性的值。
    • [[Value]]：包含这个属性的数据值。
    • 直接在对象上定义的属性前三项值都默认为true， [[Value]]特性则被设定为特定值。
    • 可以通过Object.defineProperty()方法修改属性默认的特性。
  2. 访问器属性
    • 访问器属性不包含数据值，包含一对可选的getter和setter函数。在读取访问器属性时，会调用getter函数，这个函数负责返回有效的值；在写入访问器属性时，会调用setter属性并传入新值，这个函数负责决定如何处理数据。
    • 访问器属性有[[Configurable]]、[[Enumerable]]、[[Get]]、[[Set]]四个特性。
    • 如果对象前面带有 _，说明该属性只能通过对象方法访问。

定义多个属性

利用Object.defineProperties()方法可以通过描述符一次定义多个属性。这个方法接收两个对象参数：第一个对象是要添加和修改其属性的对象，第二个对象的属性与第一个对象中要添加或修改的属性一一对应。

读取属性的特性

利用Object.getOwnPropertyDescriptor()方法，可以取得给定属性的描述符。这个方法接收两个参数：属性所在的对象和要读取其描述符的属性名称，返回值是一个对象。

创建对象

用Object构造函数或对象字面量都可以创建单个对象，但使用同一个接口创建很多对象会产生大量重复的代码。

工厂模式

函数createPerson()能够根据接受的参数来构建一个包含 所有必要信息的Person对象。该函数可以无数次调用，每次它都会返回一个包含三个属性一个方法的对象。
缺点：工厂模式解决了创建多个相似对象的问题，但无法知道一个对象的类型。

构造函数模式

原生构造函数：Object、Array，运行时会自动出现在执行环境中。
可以创建自定义的构造函数，即自定义对象类型的属性和方法。

构造函数应该以大写字母开头。
使用new操作符创建了Person的两个新实例对象，这两个对象都有一个constructor属性，该属性指向Person，即
alert(person1.constructor == Person); //true。
该例子中创建的对象既是Person的实例，也是Object的实例，因为所有对象都继承自Object。可以用instanceof来检测。

  1. 使用new操作符创建对象的步骤：
    （1）创建一个新对象
    （2）将构造函数的作用域赋给新对象
    （3）执行构造函数中的代码
    （4）返回新对象
  2. 构造函数作为普通函数调用时，属性和方法都会添加给window对象，因此可以在调用完函数时，通过window对象来调用sayName()方法。
    也可以使用call()或者apply()在某个特殊对象的作用域中调用Person()函数，将属性和方法赋给特殊对象。
  3. 构造函数的缺点：
    每个方法都要在每个实例上重新创建一遍。因为在ECMAScript中函数也是对象，因此每定义一个函数，就实例化了一个对象。这样会导致不同的作用域链和标识符解析。因此，不同实例上的同名函数是不相等的。

原型模式

每个创建的函数都有一个prototype属性，这个属性是一个指针，指向一个包含可以由特定类型的所有实例共享的属性和方法。

prototype是通过调用构造函数而创建的那个对象实例的原型对象。
优点：所有对象实例共享它所包含的属性和方法。不必在构造函数中定义对象实例的信息，直接将所有信息添加到原型对象上。
构造函数变成了空函数，但仍可以通过调用构造函数来创建新对象，而且新对象具有相同的属性和方法。

  1. 理解原型对象
    • 只要创建一个新函数，那么该函数就会被创建一个prototype属性，指向函数的原型对象。所有原型对象都会自动获得一个constructor属性，这个属性是一个指向prototype属性所在函数的指针，即
      A.prototype.constructor == A。
    • 当调用构造函数创建一个新实例后，实例内部会包含一个指针 [[Prototype]]，指向构造函数的原型对象，该指针在外部访问不到。但每个实例对象上支持一个属性proto，可以访问。且该属性连接的是实例与构造函数的原型对象，即
      person1._proto_ == Person.prototype。
    • 可以用isPrototypeOf()方法来确定对象之间是否存在这种关系，例如
      alert(Person.prototype.isPrototyprOf(person1)); //true。
    • 可以用Object.getPrototypeOf()方法获得一个对象的原型，例如
      alert(Object.getPrototypeOf(person1) == Person.prototype); //true。
    • 当代码读取某个对象的属性时，会执行搜索。首先搜索实例本身，如果找到了具有给定名字的属性，则返回该属性值；如果没有找到，则搜索该实例的原型对象，找到了则返回该属性的值。
      当为实例添加一个属性时，该属性会屏蔽原型对象中保存的同名属性，即这种行为会阻止我们访问原型中的那个属性，但不会修改那个属性。使用delete操作符可以删除实例属性，使它能够重新访问原型中的属性。
    • 使用hasOwnProperty()方法可以检测一个属性是存在于实例中还是原型中，如果存在于实例中，则返回true。
  2. 原型与in操作符
    有两种方式使用in操作符，单独使用和在for-in循环使用。
    • 单独使用时，in操作符会在通过对象能够访问给定属性时返回true，无论该属性存在于实例中还是原型中。例如：
      alert("name" in person1); //只要实例person1或其原型中包含属性name，就返回true。
    • 在使用for-in循环时，返回的是所有能够通过对象访问的、可枚举的属性，既包括存在于实例中的属性，也包括存在于原型中的属性。
      要取得对象上所有可枚举的实例属性，可以用Object.keys()方法。该方法接收一个对象作为参数，返回一个包含所有可枚举属性的字符串数组。
      要取得对象上所有实例属性，无论是否可枚举，可以使用Object.getOwnPropertyNames()方法。（constructor属性不可枚举，但仍可使用这种方法取得。）
  3. 更简单的原型语法
    前面例子中，每添加一个属性和方法就要敲一遍Person.prototype。为了更好地封装原型，可以通过对象字面量的方式来为原型添加属性和方法。
    但通过对象字面量方式创建的原型对象，其constructor属性不再指向Person，而是指向Object。也可以通过在对象字面量中设置 constructor: Person 的方式使它仍然指向Person，但这样会导致constructor属性成为可枚举属性。
  4. 原型的动态性
    实例与原型之间具有松散连接关系。实例与原型之间的连接是一个指针，而不是副本。所以对原型对象的任何修改都能从实例上反映出来。但是如果通过对象字面量的方式重写整个原型对象则会导致该实例无法找到原型。
  5. 原生对象的原型
    所有原生引用类型（Object、Array、String等）都在其构造函数的原型上定义了方法。
  6. 原型对象的缺点
    原型中所有属性都是被所有实例共享的。对于包含引用类型值的属性，通过实例修改属性会导致原型上的属性也发生变化。

组合使用构造函数模式和原型模式

构造函数模式用于定义实例属性，原型模式用于定义方法和共享的属性。这样，每个实例都会有自己的一份实例属性的副本，同时又共享着对于方法的引用，最大限度地节省了内存。

动态原型模式

动态原型模式将所有信息封装在构造函数中，通过在构造函数中初始化原型，保持了同时使用构造函数和原型的优点。

当sayName()方法不存在时，会将它添加到原型中。因此这一小段代码只会在初次调用构造函数时执行。

寄生构造函数模式

寄生构造函数模式基本思想：创建一个函数，该函数的作用是封装创建对象的代码，然后返回新创建的对象。

这个模式下，除了使用new操作符并把使用的包装函数称为构造函数之外，与工厂模式没有区别。但是在寄生构造函数模式下，返回的对象与构造函数或者与构造函数的原型属性之间没有关系，因此不能用instanceof操作符来确定对象类型。

稳妥构造函数模式

稳妥对象概念：指的是没有公共属性，而且其方法也不引用this的对象。
稳妥构造函数与寄生构造函数类似，但有两点不同：一是新创建对象的实例方法不引用this，二是不使用new操作符调用构造函数。

继承

原型链

利用原型链实现继承，基本思想是利用原型让一个引用类型继承另一个引用类型的属性和方法。每个构造函数都有一个原型对象，原型对象都包含一个指向构造函数的指针，而实例都包含一个指向原型对象的内部指针。可以让原型对象等于另一个类型的实例，此时原型对象将包含一个指向另一个原型的指针，相应地另一个原型中包含着一个指向另一个构造函数的指针。这样层层递进，就构成了实例与原型的链条。
实现原型链的一个基本模式：

在通过原型链实现继承的情况下，搜索过程沿着原型链继续向上。例如，本例中调用instance.getSuperValue()会经历三个搜索步骤：1）搜索实例；2）搜索SubType.prototype；3）搜索SuperType.prototype。

  1. 所有引用类型都默认继承了Object，因此默认原型都会包含一个内部指针，指向Object.prototype。
  2. 可以通过两种方式来确定原型和实例之间的关系。
    • instanceof操作符，只要用这个操作符测试实例与原型链中出现过的构造函数，都会返回true。
    • isPrototypeOf()方法，只要原型链中出现过的原型，都可以认为是该原型链派生的实例的原型。
  3. 给原型添加方法的代码要放在替换原型的语句之后。并且不能使用对象字面量创建原型方法，否则会重写原型链。
  4. 原型链的缺点：
    • 包含引用类型值的原型属性会被所有实例共享，当通过实例修改属性时，所有实例的对应属性都会被修改。
    • 在创建子类型的实例时，不能向超类型的构造函数中传递参数。

借用构造函数

为了解决原型中包含引用类型值所带来的问题，可以使用借用构造函数。基本思想是：在子类型构造函数内部调用超类型构造函数。
将前文例子中的SubType.prototype = new SuperType();替换为

通过使用call()方法（或apply()方法），在新创建的SubType实例环境下调用了SuperType构造函数。这样会在新SubType对象上执行SuperType()函数定义的所有对象初始化代码，这样SubType的每个实例会拥有引用类型属性的副本。即通过实例修改原型属性时，其他实例对应的属性不会再被修改。

  1. 借用构造函数也可以在子类型构造函数中向超类型构造函数传递参数。
  2. 借用构造函数缺点： 方法都在构造函数中定义，因此不存在函数复用。而且在超类型的原型中定义的方法，对子类型不可见。

组合继承

将原型链和借用构造函数的方式结合，使用原型链实现对原型属性和方法的继承，通过借用构造函数实现对实例属性的继承。

两个不同的SubType实例既分别拥有自己的属性，又可以使用相同的方法。
组合继承避免了原型链和借用构造函数的缺陷，融合了它们的优点，成为最常用的继承方式。

原型式继承

通过Object.create()方法实现原型式继承，该方法接收两个参数：一个用作新对象原型的对象和一个为新对象定义额外属性的对象。当只传入一个参数时，Object.create()方法与object()方法行为相同。

类似于原型模式，包含引用类型值的属性始终会共享相同的值。

寄生式继承

寄生式继承基本思想：创建一个仅用于封装继承过程的函数，该函数在内部以某种方式来增强对象，最后再返回对象。

寄生组合式继承

组合继承的缺点：调用两次超类型构造函数。一次是在创建子类型原型的时候，一次是在子类型构造函数内部。这样会导致在新对象上创建实例属性时会屏蔽原型中的两个同名属性。
寄生组合式继承可以解决这个问题。即借用构造函数来继承属性，通过原型链的混成形式来继承方法。

语法

  1. JS中的变量、函数名和操作符都区分大小写。
  2. 标识符格式规则：第一个字符必须是字母或_或$，其他字符可以是字母、_、$或数字。且标识符不能为关键字或保留字。
  3. 单行注释以两个斜杠 // 开头。块级注释以 /*开头，以*/结尾。
  4. 严格模式：在脚本顶部添加 "use strict"，将在整个脚本中启用严格模式；在函数内部的第一句添加该代码，也可以指定函数在严格模式下执行。严格模式下，会对某些不安全的操作抛出错误。

变量

  1. 定义变量时，用var后跟一个变量名，例如：var message。但此时message未经初始化，其值和类型都会被保存为undefined。
  2. 如果在定义变量时将其初始化为一个字符串，在修改变量时仍可以修改其类型，再次将变量赋值为数字等。
  3. 用var操作符定义的变量仅成为该作用域的局部变量。例如在函数中定义的变量，在函数退出后就会被销毁。
  4. 可以用一条语句定义多个变量，中间用 , 隔开。

数据类型

Javascript共有六种数据类型：Undefined、Null、Number、Boolean、String和Object。

typeof 操作符

  1. 对一个值使用typeof操作符可能返回以下几种字符串：
    undefined、boolean、string、number、object（值是对象或null）、function（值是函数）。

Undefined类型

  1. Undefined类型只有一个值，为undefined。
  2. 调用未经声明的变量会报错，调用未经初始化的变量会返回undefined。
  3. 对未经声明和未经初始化的变量使用typeof操作符都会返回undefined。

Null类型

  1. Null类型只有一个值，为null。
  2. Null在逻辑上表示空对象指针，使用typeof操作符检测会返回object。
  3. 判断undefined和null的相等性 null == undefined 时，返回true。

Boolean类型

  1. Boolean类型有true和false两种值。
  2. 任何类型的值调用 Boolean() 函数，都会返回一个Boolean值。如果值为 false、空字符串、0、NaN、null和undefined，则返回false，其他则返回true。

Number类型

  1. 对变量赋值时，十进制正常表示，八进制第一位为 0 ，十六进制前两位为 0x。进行算术计算时，所有数值都会被转换成十进制数。
  2. 浮点数值的保存空间是整数值的两倍，因此会合理转换。
  3. 浮点数值的最高精度是17位小数。
  4. Number.MAX_VALUE 和 Number.MIN_VALUE保存了JS能表示的最大数值和最小数值。isFinite() 函数确定了一个数值是不是有穷的。
  5. NaN用于表示一个本来要返回数值的操作数未返回数值的情况。
    • 任何涉及NaN的操作都会返回NaN。
    • NaN与任何值都不相等，包括NaN本身。
    • isNaN() 函数的参数若不能被转换为数值，则返回true，否则返回false。
  6. 非数值转换为数值的函数：Number()、parseInt()、parseFloat()。
    • 对非空且非数字的字符串使用Number()函数，会被转换为NaN。
    • parseInt() 转换字符串时，如果第一个字符不是数字字符或者负号，则返回NaN；如果第一个字符是数字，则会继续解析，直到解析完或遇到非数字字符（舍弃从该非数字字符开始的所有字符）。
      parseInt() 函数可以在第二个参数指定基数，按规定进制解析字符。
    • parseFloat() 函数转换字符串时，遇到的第一个小数点有效，因此可以解析浮点数，但只能解析十进制。

String类型

  1. 字符串可以由双引号 " 或单引号 ' 表示。
  2. toString() 方法可以将一个值转换为字符串。该方法适用于数值、布尔值、对象和字符串值。但null和undefined没有这个方法。
  3. String() 函数能够将任何类型的值转换为字符串，对于null和undefined会直接返回其字面量。

Object类型

  1. 对象可以通过执行new操作符加要创建的对象类型的名称来创建。
  2. Object类型是所有其实例的基础，即Object类型所具有的任何属性和方法都同样存在于更具体的对象中。

操作符

一元操作符

  1. 前置递增递减操作符：变量的值在语句被求值之前改变。
    后置递增递减操作符：递增和递减操作在语句被求值之后改变。
  2. 一元加减操作符：对非数值的字符串使用时转变为NaN；对对象使用时先调用它们的valueOf()和（或）toString()方法，再进行转换得到值；对其他类型使用时转换得到相应的值。
    一元加操作符对数值使用时不会有任何变化。
    一元减操作符对数值使用时将数值变为负数。

位操作符

  1. 按位非：~ 表示，相当于操作数的负值减1。
  2. 按位与：& 表示，数值对应位都是1为1，其他为0。
  3. 按位或：| 表示，数值对应位都是0为0，其他为1。
  4. 按位异或：^表示，数值对应位相同为0，相异为1。
  5. 左移：<<表示，将数值按二进制移动相应位数，不影响符号位。
  6. 右移：>>表示，保留符号位，并用符号位的值填充空位。
  7. 无符号右移：>>>表示，符号位一起右移，并用0填充空位。

布尔操作符

  1. 逻辑非：! 表示，将操作数转换为布尔值之后求反。!! 等价于 Boolean()。
  2. 逻辑与：&& 表示，短路操作符。如果左侧是false，则不会再对右侧求值，直接返回false；如果左侧是true，则返回右侧。
  3. 逻辑或：|| 表示，短路操作符。如果左侧是true，则不会再对右侧求值，直接返回false；如果左侧是false，则返回右侧。

乘性操作符

  1. 乘法：* 表示，先调用Number() 将操作数转换为数值，再进行运算。
    • 如果有一个操作数为NaN，结果是NaN。
    • 如果Infinity与0相乘，结果是NaN。
    • Infinity与非0数相乘，结果是Infinity或-Infinity，取决于非0数的符号。
    • Infinity与Infinity相乘，结果是Infinity。
  2. 除法：/ 表示，先调用Number() 将操作数转换为数值，再进行运算。
    • 如果有一个操作数是NaN，结果是NaN。
    • 如果Infinity被Infinity除，结果是NaN。
    • 如果零被零除，结果是NaN。
    • 非零的有限数被零除，结果是Infinity或-Infinity，取决于非0数的符号。
    • Infinity被任何非0数值除，结果是Infinity或-Infinity。
  3. 求模：% 表示，执行除法运算，返回余数。

加性操作符

  1. 加法：+ 表示。如果两个操作数都是数值，执行常规加法。若其中一个操作数是字符串，则执行字符串的拼接。
  2. 减法：- 表示。先调用Number() 将操作数转换为数值，再进行运算。若其中一个操作数为NaN，结果为NaN。

关系操作符

>、<、>=、<=四个关系操作符。

  1. 其中一个为数值时，将另一个转换为数值比较。
  2. 若两个都是字符串，则比较编码值（大写字母的字符编码都小于小写字母）。
  3. 若其中一个为NaN，结果为NaN。

相等操作符

  1. 相等==和不相等!=：先转换类型再进行比较。
    null == undefined为true。
    NaN == NaN为false。
  2. 全等===和不全等!==：仅比较而不转换类型，需考虑类型是否相同。
    null === undefined为false。

条件操作符

  1. variable = boolean_expression ? true_value : false_value
    若问号前表达式的值为true，则返回第一个值，否则返回第二个值。

赋值操作符

  1. 用 = 表示。

逗号操作符

  1. 用 , 表示，可以用于声明多个变量。

语句

  1. if语句：条件语句
  2. do-while语句：在对条件表达式求值之前，循环体内的代码会至少执行一次。
  3. while语句：先对条件表达式求值，再决定是否执行循环体内的代码。
  4. for语句：for (initialization; expression; post-loop-expression) statement。可以在执行循环之前初始化变量，同时定义循环条件和循环后执行的代码。
    注意：由于ECMAScript不存在块级作用域，for循环定义的变量可以在外部访问到。
  5. for-in语句：可以用来枚举对象的属性。
  6. label语句：label: statement。在代码中添加标签，该标签可以在将来由break或continue语句引用。
  7. break和continue语句：break语句退出循环后执行循环之后的语句；continue退出循环后从循环的顶部继续执行。
  8. with语句：with (expression) statement。将代码的作用域设置到一个特定对象中。
  9. switch语句：对每个case后面添加一个break语句，避免同时执行多个case代码的情况。

函数

  1. 函数声明：function functionName (arg0, arg1, … , argN) { statements }
  2. 若函数中包含return，则函数会在执行完return语句后立即退出。
  3. 如果定义了命名参数，传入时参数个数可以与命名参数个数不同。
    声明函数时可以不包含命名的参数，并在函数内部用arguments对象来访问参数数组。
    命名参数和arguments对象可以共存，但arguments的值与对应命名参数的值保持同步。
  4. 如果定义了两个名字相同的函数，则后面的函数会覆盖前面的函数。

<script>元素

  1. 使用<script>元素的方式有两种：直接在页面中嵌入Javascript代码和包含外部Javascript文件。
    • 嵌入Javascript代码
      – 须为<script>指定type属性 type=”text/javascript” 。
      – 代码块中不能出现 字符串，否则直接读为结束标签，报错。可以在代码块中通过转义字符“\” 解决这个问题，写为 <\/script>。
    • 包含外部Javascript文件
      – 必须包含src属性，可以是外部域文件。
      – 在带有src属性的标签内嵌入的代码会被忽略。
  2. 浏览器显示页面时，遇到<body>标签才会开始呈现内容。因此为了不影响浏览器正常加载页面的速度，将<script>标签放在<body>标签的最后。
  3. 为<script>标签添加defer属性后，其包含的脚本将会延迟到整个页面解析完毕后再运行。该属性只适用于外部脚本文件。
  4. 为<script>标签添加async属性后，浏览器会立即下载文件，不保证按照指定顺序执行，但会在页面的load事件前执行。该属性只适用于外部脚本文件。

文档模式

  1. 标准模式下，可以用<!DOCTYPE html>来开启一个HTML5文档。
  2. 如果文档开始处没有文档类型声明，则所有浏览器会默认开启混杂模式。

<noscript>元素

  1. 在浏览器不支持脚本或脚本被禁用时，显示<noscript>元素中的内容，用以向用户提示。

github+hexo搭建个人博客以及心酸踩坑史

发表于 2019-01-23 | 分类于 动动手

最近一直想要拥有个自己的地盘，花了一些时间，踩了很多坑，才终于搭好了个人博客。
那么第一篇博客就用来记录一下这个艰辛的过程啦。

准备工作

安装Git for Windows

选择适合自己的版本下载 Git for Windows ，安装时一路点击next。
安装好后打开cmd，输入命令

git version

看到下图说明安装完成。


安装完成后会在文件夹中点击鼠标右键时出现 Git GUI Here 和 Git Bash Here 的选项。

安装Node.js

Node.js下载地址
安装时选择Add to Path，继续点击next，不需要配置环境变量，且安装Node.js时将会自动安装npm。

安装完成后打开cmd，输入命令

node -v
npm -v

看到下图说明安装完成。

安装Hexo

以管理员身份进入cmd，输入以下命令安装

npm install hexo-cli -g
npm install hexo-deployer-git --save

（-g的含义为安装到全局目录中）

安装完成。测试hexo是否完成安装，输入命令

hexo -v

出现版本号为正确安装，如下图。

（此处我一jio踩了第一个大坑，详情见后文）

本地创建放置博客的文件夹

我创建的位置为 “E:\github\blog” 。

进入文件夹，点击鼠标右键，打开 Git Bash Here ，对文件夹进行初始化，输入命令：

hexo init

执行命令后，文件夹中出现如下目录：

接下来输入命令

hexo generate //也可写作hexo g，生成静态页面
hexo server //也可写作hexo s，打开本地服务器

成功打开本地服务器后，显示下图

此时可登录 localhost:4000，查看效果图如下

这样就说明hexo部分初始化成功啦。

将博客部署到Github

创建博客repository

到 Github官网 注册Github账号。
创建new repository。

注意：创建repository时，name必须为“你的github用户名.github.io”，否则后面打开页面时为404。

（这里我一开始是随便起的名字，打开页面一直显示404，后来发现要对已经创建的repository进行rename之后才能打开页面）

创建ssh密钥

右键打开 Git Bash ，输入命令

ssh-keygen -t rsa -C "your_email@example.com"

这里邮箱填写Github登录邮箱，出现以下文字

Generating public/private rsa key pair.
Enter file in which to save the key (/c/Users/xin/.ssh/id_rsa): [Press enter]

可以选择直接按enter将文件保存为括号中的默认名字，也可以按照格式修改id_rsa部分。

接着根据提示需要输入密码并确认密码，这里我选择无密码，直接按两次enter。完成后显示：

在github账户中添加公钥

进入文件夹 “c:/users/xin/.ssh” 中，用记事本打开 id_ras.pub ，复制内容。

打开Github，点击头像→点击settings→点击SSH and GPG keys。

点击新建一个ssh keys，名字随意，在下面粘贴刚刚复制好的公钥，点击add新建成功。

回到本地，在 Git Bash 中输入命令：

git config --global user.name "zhaoxin0178" //这里用户名为github用户名
git config --global user.email "your_email@example.com" //填写github登录邮箱

然后输入下面命令，测试ssh是否设置成功

ssh -T git@github.com //本条命令不要修改任何内容

出现下图时，输入yes

接下来可以看到出现了你的github用户名，说明配置成功。

将博客部署到Github

在创建的博客文件夹中打开hexo配置文件 _config.yml ，将文件最后修改为（注意 _config.yml 文件中所有”:”后面都要加一个空格再填具体内容）

deploy:
type: git
repository: https://github.com/zhaoxin0178/zhaoxin0178.github.io
branch: master

（repository地址为你的github项目地址）

写博客

在创建的本地博客文件夹中右键打开 Git Bash ，输入

hexo new post “The first blog”

输出下图，说明博客创建成功，The-first-blog.md 就是我们要编辑的博客正文。


Markdown文件在windows上可以用 MarkdownPad 打开。
编写完博客后，在博客文件夹根目录下打开 Git Bash ，输入以下命令

hexo g //生成静态网页
hexo s //本地预览效果
hexo d //上传到github

这时打开域名xxx.github.io就可以看到你的博客啦。
（这里顺便推荐一下windows图床工具 PicGo，支持拖拽上传图片并且可以直接生成md格式的连接，在博客中插入图片就比较方便了~）

主题配置

我看了一些主题，还是不得不觉得Next最火是有理由的！下面就配置一下Next。
在博客文件夹中右键打开 Git Bash ，输入

git clone https://github.com/iissnan/hexo-theme-next themes/next


得到克隆的next主题文件。
打开 E:\github\blog_config.yml ，修改：

theme: next

并修改主题语言为中文，修改博客名字等

title: Cetus' Blog
subtitle:
description:
keywords:
author: Cetus
language: zh-Hans
timezone: Asia/Shanghai

这里timezone必须这样填，不要填其他地址，否则后面会报错 Cannot read property ‘offset’ of null。
这样主题就设置好了。
Next有几种不同主题，修改方法如下：
打开 E:\github\blog\themes\next_config.yml 文件，按 ctrl+f 找到 scheme，将想要选择的主题取消掉注释，并把其他主题注释掉就可以了。

更改域名

购买域名

可以到 万网 购买。
注册域名时需实名制，否则后面可能一直无法正确解析域名。
在 E:\github\blog\source 下新建文件，命名为 CNAME，没有后缀。

创建好后用记事本打开，将购买的域名填写进去，比如我要填写 zhaoxin.site。

在博客文件夹右键打开 Git Bash ，输入

hexo g -d

上传到github。

配置DNS

登录阿里云，点击”控制台→域名→我的域名→修改DNS”，将DNS修改为

f1g1ns2.dnspod.net
f1g1ns1.dnspod.net

打开 DNSPOD，注册后添加域名，如下图

点击 zhaoxin.site，进入后点击添加记录。

添加两条，主机记录分别为 @ 和 www，记录类型都为A，记录值为之前创建的github博客页面的ip地址。

打开cmd命令，ping一下你的页面（例如我是zhaoxin0178.github.io）可以得到ip地址。

添加完成后，等待生效，最迟72小时可以通过已购域名访问博客页面。（其实不用等那么久，大概1-2小时就可以了~）

踩坑记录

测试hexo是否安装好时出现 Command not fond

选择高级系统设置→环境变量→path。

重新测试，出现版本号，说明安装成功了。

记事本打开 E:\github\blog_config.yml 后选择另存为，保存时选择文档格式为UTF-8，再替换掉原 _config.yml 文件即可。
